Deploying My .NET Blog to Render

Recently, I set out to publish my personal .NET blog using Render, a platform I hadn’t used before but one that looked appealing for simple web hosting. Things didn’t go entirely smoothly. Here’s a recap of what went wrong, what I learned, and what eventually got me over the finish line.

My Goal

I’d built a simple ASP.NET Core MVC blog project with a controller to serve my blog posts from HTML files. All I wanted was to get it hosted somewhere easily and Render seemed like a good option. It was, but only after some trial and error.

Attempt 1: Native .NET - Static Site

Initially, I followed Render’s static Web Service guide. The default build command I was using looked like this:

dotnet publish MyBlog/MyBlog.csproj -c Release -o out

But this failed with:

bash: line 1: dotnet: command not found

Attempt 2: Docker Deployment - Dynamic Site

I wrote a Dockerfile that used the official .NET 8 SDK image, published the app, and exposed port 80

I also added a render.yaml file to tell Render how to build and deploy the app using Docker. Once this was in place, Render successfully built and ran the app. But.....

A Mysterious "Not Found"

Even though the app deployed, if i visited the URL i just got a "Not Found" page. It wasn’t a 404 page from my app — it was Render’s own page.

After digging in, I realized I’d left in some non-standard code from a .NET 9 preview template I’d previously experimented with. Once I removed them and stuck with the conventional middleware setup (UseStaticFiles(), MapControllerRoute(), etc.), everything worked as expected.

Success!

After cleaning up the Docker file and reverting my Program.cs to the basic methods the site deployed correctly - I finally saw something other than “Not Found”.

TL;DR: If you're deploying a .NET app to Render, use Docker, stick with .NET 8, and use basic syntax unless you’re absolutely sure it will work!
