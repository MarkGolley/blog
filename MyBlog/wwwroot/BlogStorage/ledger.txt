# Learning Ledger - Mark

This file tracks concepts, patterns, APIs, and language features I've demonstrated understanding of, so the agent can narrow its teaching over time and I can see my progress over the year.

**Owned by:** the agent (auto-updated). I can hand-edit too (e.g. demote something I've forgotten, or add a concept I learned outside a session).

**How statuses work:**
- **Mastered** - I explained it back correctly, on more than one occasion, and I'm confident I could defend it in PR review without help. Agent should not re-explain unless I ask. May still mention by name.
- **Proficient** - Explained correctly at least once, recently. Agent can skip the long definition but should still drop a one-line refresher when used.
- **Learning** - Introduced and partially understood. Agent must still explain in full when it appears.
- **Parked** - Came up once, deferred (TODO / out of scope). Not on the active learning path.
- **Stale** - Was Mastered/Proficient but I've recently struggled with it. Agent should re-teach as if Learning.

**How the agent updates this file:**
- After a passed Gate 1 or Gate 2 on a concept, promote it (Learning -> Proficient -> Mastered).
- After a "I don't understand" or a wrong gate answer on a previously promoted concept, demote it to Stale.
- After honouring a `SHIP IT NOW` override, do NOT promote anything in the ledger - overrides are understanding-debt, not understanding.
- New entries: append with today's date and a one-line "evidence" note (what I demonstrated).
- Never silently delete entries. Demote, don't drop.
